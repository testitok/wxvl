#  微软发布 Windows 11 开箱即用热补丁修复 RRAS 远程代码执行漏洞  
会杀毒的单反狗
                    会杀毒的单反狗  军哥网络安全读报   2026-03-16 01:00  
  
**导****读**  
  
  
  
微软发布一项带外 (OOB) 更新，以修复影响 Windows 11 企业版设备的安全漏洞。这些设备接收的是热补丁更新，而不是常规的“周二补丁日”累积更新。  
  
![](https://mmbiz.qpic.cn/mmbiz_png/PaFY6wibdwyIP93RPd3rFWHNCfxUPrsknG1jBEnLoxgCibODfRykFyo0rnicuHgr7FibSmULC1mAmIZDJeXOKiaJ5OaKuvJEuFXEibun3wvAy0Nnw/640?wx_fmt=png&from=appmsg "")  
  
  
KB5084597热补丁更新于昨日发布，旨在修复 Windows 路由和远程访问服务 (RRAS) 管理工具中的漏洞，这些漏洞可能允许在连接到恶意服务器时执行远程代码。  
  
  
微软发布的一份安全公告称：“微软已发现 Windows 路由和远程访问服务 (RRAS) 管理工具中存在安全问题，该问题可能允许在连接到恶意服务器时执行远程代码。”  
  
  
“这个问题仅适用于企业客户端设备运行热补丁更新并用于远程服务器管理的有限场景。”  
  
  
KB5084597 更新适用于 Windows 11 版本 25H2 和 24H2，以及 Windows 11 企业版 LTSC 2024 系统。  
  
  
微软表示，此次热补丁修复的漏洞编号为CVE-2026-25172、CVE-2026-25173和CVE-2026-26111，这些漏洞已在2026 年 3 月的“补丁日”更新中得到修复。  
  
  
“通过域认证的攻击者可以利用此漏洞，诱骗已加入域的用户通过路由和远程访问服务 (RRAS) 管理单元向恶意服务器发送请求。”这三个缺陷的描述中写道。  
  
  
该公司表示，此次热补丁更新是累积性的，包含了 3 月 10 日发布的 2026 年 3 月 Windows 安全更新中的所有修复和改进。  
  
  
虽然这些漏洞已在“补丁日”修复，但安装累积更新需要重启设备。然而，有些设备用于运行关键任务应用程序和服务，这些设备无法轻易重启。  
  
  
为了保护这类设备，热补丁更新通过对正在运行的进程进行内存修补来应用新的漏洞修复程序。同时，它们还会更新磁盘上的文件，以便设备下次重启时，这些修复程序仍然有效。
